## 1. Datasets

### 1.1 Dataset A
- **Файл:** `S07-hw-dataset-01.csv`
- **Размер:** 12000 строк, 8 признаков.
- **Признаки:** Числовые.
- **Пропуски:** Нет.
- **"Подлости":** Разные масштабы признаков (требует StandardScaler).

### 1.2 Dataset B
- **Файл:** `S07-hw-dataset-02.csv`
- **Размер:** ~8000 строк, 2 признака + шум.
- **Признаки:** Числовые.
- **Пропуски:** Нет.
- **"Подлости":** Нелинейная форма ("луны"), наличие шума.

### 1.3 Dataset C
- **Файл:** `S07-hw-dataset-04.csv`
- **Размер:** 500 строк, 32 признака.
- **Признаки:** Смешанные (числовые + категориальные).
- **Пропуски:** Есть (в числовых признаках).
- **"Подлости":** Пропуски, категории, высокая размерность.

## 2. Protocol
- **Препроцессинг:** Использован `ColumnTransformer` с `StandardScaler` и `SimpleImputer(strategy='median')` для чисел, и `OneHotEncoder` для категорий. `sample_id` исключен из признаков.
- **Поиск гиперпараметров:** Для KMeans перебор $k \in [2, 10]$ по метрике силуэта. Для DBSCAN — поиск `eps`.
- **Метрики:** Silhouette, Davies-Bouldin, Calinski-Harabasz.
- **Визуализация:** PCA(2D) для отображения результатов.

## 3. Models
- **KMeans:** фиксированные `random_state=42`, `n_init=10`.
- **DBSCAN:** подбор `eps` и `min_samples`.
- **AgglomerativeClustering:** использован для сравнения на DS2.

## 4. Results

### 4.1 Dataset A
- **Лучший метод:** KMeans (k=3).
- **Метрики:** Silhouette ~0.68, DB ~0.52, CH ~14500.
- **Комментарий:** Идеальное разделение на сферы после масштабирования.

### 4.2 Dataset B
- **Лучший метод:** AgglomerativeClustering (k=2).
- **Метрики:** Silhouette ~0.45.
- **Комментарий:** Превосходит KMeans на нелинейных данных.

### 4.3 Dataset C
- **Лучший метод:** KMeans (k=4).
- **Метрики:** Silhouette ~0.22.
- **Комментарий:** Отражает структуру, несмотря на пропуски и категории.

## 5. Analysis

### 5.1 Сравнение алгоритмов (важные наблюдения)
KMeans не справляется с нелинейными формами (DS2). Масштабирование критично для DS1. В DS4 OneHotEncoder позволил учесть категории.

### 5.2 Устойчивость (обязательно для одного датасета)
Проведена проверка KMeans на DS1 (5 запусков с разными random_state). Средний ARI = 1.0. Решение абсолютно устойчиво.

### 5.3 Интерпретация кластеров
Кластеры в DS1 четко разделяют группы по интенсивностям признаков. В DS4 кластеры частично коррелируют с категориальным признаком `cat_a`.

## 6. Conclusion
